# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Narratio, please report it responsibly using [GitHub's private security advisory feature](https://github.com/your-repo/narratio/security/advisories/new).

**Please do not open a public issue for security vulnerabilities.**

## Scope

The following are in scope for security reports:

- The Narratio web application and API
- Authentication or authorization issues
- Dependency vulnerabilities
- Data exposure or injection flaws

## Out of Scope

- The hosted SQLite database content (article headlines from public news APIs)
- Denial-of-service attacks
- Social engineering

## Response

This is a personal project maintained in spare time. I'll do my best to:

- Acknowledge reports within **7 days**
- Provide a fix or mitigation within **30 days** for confirmed issues

## Supported Versions

Only the latest version on the `main` branch is supported.
